# blog-it

기술 작업 한 번 → 블로그 초안 + 이력서 업데이트 한 번에.

**Announce at start:** "blog-it 스킬로 블로그 초안과 이력서 항목을 작성하겠습니다."

## 워크플로우 (4단계)

### 1단계: 정보 수집 (PAAR 구조)

사용자가 `/blog-it`와 함께 작업 내용을 자유 텍스트로 전달합니다.
다음 정보가 명시되지 않았다면 **반드시 추가 질문**. PAAR 순서로 채웁니다:

- **P (Problem/Project)**: 기능명/프로젝트명, 어느 앱, 해결해야 했던 문제·배경
- **A (Action)**: 구체적으로 취한 접근, 사용 기술, 본인의 역할 (시도했지만 실패한 것 포함)
- **A (Analysis, 선택)**: 왜 이 방법이 통했는가, 트레이드오프, 다른 방법 대비 장단점
- **R (Result)**: 정량적 성과 (개선 수치, 지표 변화, 처리 시간 등)
  - 수치가 전혀 없다면 **한 번 더 질문**: "측정 가능한 수치(Before/After, 처리 건수, 시간)가 있을까요?"
  - 정말 수치가 없을 때만 사용자 반응·배운 점으로 대체
- **코드 스니펫이 있나** (있으면 어느 부분이 핵심인지)

질문은 **한 번에 3개 이하**로 묶어서. 정보가 충분하면 바로 2단계.

### 2단계: 두 초안 동시 생성

#### A. 블로그 마크다운 초안

- 한국어, 약 2500-3500자 (중간 길이)
- 톤은 내용에 맞춰 (트러블슈팅 → 스토리텔링, 튜토리얼 → 단계별 등)
- 자세한 스타일 가이드: `references/blog-template.md` 참조
- 코드 블록은 언어 명시 필수

#### B. 노션 이력서 추가 항목

- 3-5줄로 간결하게
- 형식 및 노션 블록 구조: `references/notion-section.md` 참조

### 3단계: 사용자 검토

두 초안을 한 응답에 모두 출력하고 질문:
- "이대로 진행할까요? 수정하실 부분 있으면 알려주세요."

수정 요청 → 반영 → 다시 보여주기. 확정 답("OK", "좋아", "진행" 등)까지 반복.
**이 단계 절대 건너뛰지 않음.**

### 4단계: 발행 (사용자 확정 후)

#### A. 마크다운 파일 저장

경로: `~/Documents/Obsidian Vault/기술 블로그/YYYY-MM-DD-{slug}.md`
- YYYY-MM-DD: 오늘 날짜
- slug: 제목을 영문 kebab-case로 변환 (한글 제목은 의미 살린 짧은 영문)
- 같은 날 같은 slug 충돌 시 `-2`, `-3` 접미사
- 경로에 공백(`Obsidian Vault`, `기술 블로그`)이 포함되므로 Bash로 다룰 때는 반드시 쌍따옴표로 감쌀 것

Write 도구로 저장 (절대 덮어쓰지 않음).

#### B. 노션 이력서 업데이트

이력서 페이지 ID: `02751b8297224773acf5d1e4f5de665b`

순서:
1. `mcp__claude_ai_Notion__notion-fetch`로 페이지 현재 상태 확인
2. "AI 에이전트" 섹션 존재 여부 확인 (헤딩 텍스트 검색)
3. **없으면 (최초 1회)**:
   - 사용자에게 "AI 에이전트 섹션이 없어서 처음 생성합니다. 위치는 Projects 섹션 다음으로 가도 될까요?" 확인
   - OK → `notion-update-page`로 헤딩 + 첫 항목 동시 추가
4. **있으면**: 섹션 마지막에 새 column 블록 append

자세한 노션 호출 패턴: `references/notion-section.md`

#### C. 기록 로그 업데이트

스킬 폴더의 `history.md` 파일 `## 기록` 섹션 **맨 아래에** 이번 작업 요약을 append:

- 양식은 `history.md` 상단의 "항목 양식 (PAAR 요약)" 참조 (PAAR 순서 엄수)
- 기존 항목은 수정하지 않고 새 항목만 추가
- Edit 도구로 마지막 항목 뒤에 이어 붙이기 (Read → 마지막 항목 문자열 찾아 그 뒤에 append)

#### D. 완료 보고

```
✅ 블로그 초안: ~/Documents/Obsidian Vault/기술 블로그/YYYY-MM-DD-{slug}.md ({글자수}자)
✅ 노션 이력서 업데이트: https://www.notion.so/02751b8297224773acf5d1e4f5de665b
✅ 기록 로그 추가: ~/.claude/skills/blog-it/history.md

다음 단계 (수동):
1. 마크다운 파일 검토 / 다듬기
2. velog (메인, 한국 SEO 강함) → Medium → 기타 순으로 붙여넣기
3. velog 발행 후 URL을 노션 항목 + history.md 양쪽에 추가하면 좋음
```

## 사용 예

```
사용자: /blog-it 오늘 Flutter Riverpod에서 Stream 메모리 누수 해결한 거 정리해줘
스킬: (필요시 추가 질문 → 두 초안 → 검토 → 저장)
```

## 주의사항

- 노션 페이지 수정 전 항상 fetch로 현재 구조 확인
- 마크다운 파일 절대 덮어쓰기 금지
- 사용자 검토 단계 절대 스킵 금지
- 첫 항목 추가 시 섹션 위치는 **반드시 사용자 확인**
- 한국어 글 중 AI 티 나는 문체 회피 (`references/blog-template.md` 회피 표현 목록 참조)
